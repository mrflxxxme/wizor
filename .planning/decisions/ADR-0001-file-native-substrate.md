## Context

Для WIZOR нужен субстрат координации ИИ-команды. Есть два полярных полюса: (а) тяжёлые оркестраторы со своими runtime'ами (claude-flow, CrewAI, LangGraph), требующие внешних сервисов и «установки», и (б) чистый файловый подход на стандартных инструментах Claude Code. Команда — founder + AI-агенты; горизонт — 12 месяцев до agency-gate. Приоритеты: низкий порог старта, прозрачность состояния, дешёвый context-recall, отсутствие vendor lock-in на оркестратор.

Проектный CLAUDE.md обязан быть самодостаточным (без зависимости от глобального ruflo-конфига пользователя).

## Decision

Харнесс WIZOR использует **файл-нативный субстрат**: агенты = Claude Code Agent tool; состояние и память = git-markdown/JSON-файлы (STATUS, HANDOFF, JOURNAL, MEMORY-INDEX, contracts, gates); все форматы описаны в Charter §8. claude-flow и AgentDB могут подключаться **опционально** как семантический индекс поверх тех же файлов, когда история разрастётся настолько, что grep перестанет справляться, — но их отсутствие никогда не блокирует работу.

## Consequences

- Нулевые внешние зависимости на старте; харнесс запускается в любой Claude Code-сессии.
- Весь контекст доступен через стандартный git — история, diff, blame.
- Агентам не нужны специальные MCP-серверы для базовой координации; JIT-загрузка по MEMORY-INDEX снижает token-расход.
- Компромисс: семантический поиск по большой истории — только через grep или опциональный AgentDB; без AgentDB нет embedding-поиска.
- При росте истории (> 300 строк JOURNAL, > 50 ADR) — включение AgentDB как ускорителя (документируется отдельным ADR).

## Alternatives considered

| Альтернатива | Pro | Contra | Почему отклонили |
|---|---|---|---|
| claude-flow mandatory | Семантический поиск, граф памяти | Внешняя зависимость, сложность старта, risk orphan state | Излишне для текущего масштаба; включается опционально позже |
| LangGraph / CrewAI | Граф состояний, retry | Python-runtime, не нативен для Claude Code | Не интегрируется без дополнительного сервера |
| CloudEvents JSON-хендоффы | Стандарт | Тяжёлые; избыточны для 8-агентной команды | Заменены компактными markdown-блоками (§8.6) |
