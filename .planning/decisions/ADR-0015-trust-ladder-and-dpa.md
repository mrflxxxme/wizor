## Context

Авто-применение изменений на чужом production-сайте — юридически и технически рискованное действие. Ошибка (порча production) = утрата доверия + потенциальный ущерб. PRD §3 закрепляет доктрину «trust ladder»: авто-правки должны завоёвывать доверие поэтапно, а не требовать его авансом. Юридически необходим DPA (договор/соглашение на обработку данных и авто-изменение) до первой авто-правки.

## Decision

Безопасность авто-правок обеспечивается четырьмя механизмами:
1. **Approval-gate по умолчанию:** каждое изменение ставится в очередь на approve; пользователь явно включает `auto-apply` по конкретному типу машиночитаемой правки (robots.txt, JSON-LD, IndexNow, llms.txt).
2. **FAQ (видимый контент) — никогда не авто-применяется** (только review/merge-flow: staging → предпросмотр → явное подтверждение). Стоячий инвариант №10 Charter §6.
3. **DPA-флоу:** авто-правки недоступны до акцепта DPA; факт акцепта хранится неизменяемо. Авто-apply деактивируется при отзыве DPA.
4. **Идемпотентность + rollback:** каждая правка обратима; 1-click rollback к предыдущему состоянию; откат логируется в **append-only audit log** (diff + timestamp + инициатор). Повторное применение = идемпотентно.

Инварианты 3 и 10 Charter §6 проверяются в каждом аудите.

## Consequences

- Пользователь видит trust ladder: начинает с approval-mode, добавляет авто-типы по мере уверенности.
- DPA = юридическая защита WIZOR + явное согласие клиента = основа для insurance/enterprise-договоров.
- Append-only audit log = доказательная база при инцидентах; не может быть изменён задним числом.
- Closed-loop attribution (Auto track) требует применённых правок; trust ladder = путь к opt-in.
- Компромисс: approval-gate замедляет time-to-fix; это сознательный выбор в пользу доверия.
- staging-предпросмотр для FAQ = дополнительная UX-сложность, оправданная рисками видимого контента.

## Alternatives considered

| Альтернатива | Pro | Contra | Почему отклонили |
|---|---|---|---|
| Auto-apply по умолчанию | Меньше friction | Риск порчи production без явного согласия; юридически уязвимо | Нарушает trust-ladder доктрину PRD |
| Только manual (нет авто вообще) | Нет рисков авто | Потеря главного дифференциатора | Уничтожает moat WIZOR |
| DPA только для enterprise | Меньше friction для SMB | Нет юр. защиты при авто-правках даже для SMB | 152-ФЗ и общий риск требуют DPA для всех |
