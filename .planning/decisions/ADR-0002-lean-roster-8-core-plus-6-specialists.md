## Context

Команде нужен ростер агентов, покрывающий полный цикл Аудит→Мониторинг→Рекомендации→Авто-имплементация→Верификация при минимуме токен-расхода. Крайности: (а) один монолитный агент-«делай-всё» — перегружен контекстом, теряет специализацию; (б) десятки агентов на каждую подзадачу — токен-перерасход на координацию. Домены WIZOR технически разнородны: AEO/GEO-наука, crawler-инженерия, LLM-роутинг, CMS-интеграция, 152-ФЗ, cloud DevOps — нужна предметная экспертиза, но не для каждой фазы.

## Decision

**8 постоянных агентов ядра** (planner/Opus, architect/Opus, backend-implementer/Sonnet, frontend-implementer/Sonnet, reviewer/Sonnet, verifier/Haiku, auditor/Opus, memory-curator/Haiku) обеспечивают полный 9-шаговый цикл. **6 профильных агентов on-demand** (geo-domain-expert, crawler-probe-specialist, llm-router-specialist, cms-connector-specialist, compliance-152fz-specialist, devops-infra-specialist) спавнятся только когда фаза касается их домена — иначе 0 токенов. Профильные работают в **mode B**: сами пишут код своего домена (не делегируют implementer'у), используя Sonnet для кодинга и эскалируя на Opus только при высокой сложности/суждении.

## Consequences

- Токены тратятся только на вовлечённых агентов; большинство фаз используют ≤4 агентов.
- Глубокая предметная экспертиза в GEO/compliance/crawler без постоянного overhead'а.
- Architect остаётся арбитром конфликтов и ADR-моментов — единственная Opus-роль с кросс-доменным мандатом.
- Компромисс: planner должен точно идентифицировать, какой специалист нужен фазе; ошибка — лишний токен-расход или пропущенная экспертиза.
- Founder = оркестратор; отдельный агент-координатор не нужен.

## Alternatives considered

| Альтернатива | Pro | Contra | Почему отклонили |
|---|---|---|---|
| Один mega-agent | Простота координации | Контекст-перегрузка; нет специализации | Нереально при 13 bounded contexts |
| 20+ micro-agents | Максимальная специализация | Взрывной координационный overhead | Токен-смерть для solo-founder операции |
| Mode A (специалист-советник, implementer пишет) | Единый стиль кода | Двойной перевод задачи; потери экспертизы | Mode B эффективнее в CMS/crawler-доменах |
