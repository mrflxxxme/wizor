## Context

В многофазной ИИ-разработке основные патологии памяти: (а) агенты грузят весь исторический контекст и упираются в token-лимит; (б) состояние фазы теряется при смене сессии; (в) разные агенты пишут в одни файлы и конфликтуют. Решение должно работать без внешних сервисов и сохранять читаемость для founder'а через git.

## Decision

Система памяти WIZOR — **файл-нативная, курируемая**:
- **MEMORY-INDEX.md** в корне `.planning/` — таблица `| tag | topic | pointer | gist | updated |`; агент читает индекс → грузит полный текст только при попадании тега.
- **Head-summary:** каждый документ >2 KB начинается с `<!-- HEAD-SUMMARY (≤500т): ... -->`; агент читает summary первым.
- **Ротация:** JOURNAL.md > 300 строк → архив `_session-context/archive/JOURNAL-YYYY-Qn.md`; STATUS.md — только rolling (история → PHASE-HISTORY.md); HANDOFF.md ≤ 2 KB снапшот.
- **memory-curator/Haiku** — единственный писатель всего состояния/памяти (STATUS, HANDOFF, JOURNAL, MEMORY-INDEX, auto-README). Остальные агенты читают, но не пишут состояние.
- **auto-README:** memory-curator регенерирует статус-блок README на шаге 8 каждой фазы.
- **AgentDB** (claude-flow) — опциональный семантический индекс поверх тех же файлов; включается при переполнении grep-базированного поиска; описывается отдельным ADR.

MUST-NOT для memory-curator: секреты, полный код, чужой контент (только ссылки).

## Consequences

- Агент при любом вопросе читает MEMORY-INDEX (~1 KB) → решает, нужен ли полный файл; типичная задача грузит 5–15 KB контекста вместо 100 KB+.
- Единственный писатель исключает конфликты состояния между агентами.
- head-summary — контракт документа: автор обязан его писать.
- Компромисс: ручная курация MEMORY-INDEX требует дисциплины; при пропуске индекс устаревает.
- Ротация JOURNAL предотвращает раздувание файла, но требует регулярного вызова memory-curator.

## Alternatives considered

| Альтернатива | Pro | Contra | Почему отклонили |
|---|---|---|---|
| AgentDB mandatory | Семантический поиск | Внешний сервис; усложняет старт | Опционален; включается по необходимости |
| Каждый агент пишет свой memory | Автономия | Конфликты; размытое состояние | Единый curator = единый источник истины |
| Нет индекса — grep по всем файлам | Простота | O(n) по всем файлам; token-перерасход | MEMORY-INDEX снижает стоимость recall |
