# 03-ESCALATION — когда и как эскалировать

## Правило 80/20

**Решай сам (80%):** если ответ однозначен из phase-spec + charter + ADR — действуй, не спрашивай.

**Эскалируй (20%):** когда решение выходит за зону мандата агента или любой из триггеров ниже.

## Территория founder — ВСЕГДА эскалировать

| Домен | Примеры |
|---|---|
| **Юридическое / 152-ФЗ** | Менять модель обработки ПДн, DPA-условия, РКН-логику |
| **Деньги / ценообразование** | Тарифы, триал-периоды, kill-switch бюджета, ЮKassa |
| **Бренд / коммуникация** | Формулировки в продукте про «гарантию видимости», тон маркетинга |
| **Security** | Обход auth-проверки, изменение audit-log, секреты в коде |
| **Архитектура** | Решение вне существующих ADR, новый bounded context, смена стека |
| **PR tier 3+ merge** | Явный approve founder обязателен — CI-green недостаточно |
| **Gate P0 → P10 auto-fix** | founder_signature обязательна в gate-файле |

## Правило ≤2 ревью-циклов

`reviewer` → `implementer` — максимум 2 цикла. Если после 2 цикла не пройдено:
1. Эскалируй к `architect`/Opus — арбитраж.
2. Если `architect` не разрешает — эскалируй founder (записи в HANDOFF.md).

## Формат эскалации

Используй во всех передачах — в HANDOFF.md, в PR-описании, в сообщении founder:

```
### ESCALATION · <phase> · <ISO-ts>

**Контекст:** <1–2 предл. что происходит и почему это важно>

**Развилка:** <суть дилеммы в одном предложении>

**Варианты:**
  A) <вариант> — плюсы: ... минусы: ...
  B) <вариант> — плюсы: ... минусы: ...

**Рекомендация агента:** <A|B|другое> — <краткое обоснование>

**Заблокировано:** <что стоит на паузе до решения>
```

## Уровни арбитража

```
Агент → architect/Opus   (кросс-контекстный конфликт, ADR-момент)
     → founder           (территория founder или 3-й цикл ревью)
     → новая сессия      (контекст > 120 KB, состояние → HANDOFF.md)
```

## Anti-patterns

- **«Сделаю как умею» при архитектурной неоднозначности** — лучше уточнить, чем переделывать.
- **Эскалировать каждое решение** — founder не должен одобрять форматирование файлов.
- **Эскалировать без подготовленных вариантов** — всегда приходи с A/B + рекомендацией.
- **Продолжать ревью-цикл > 2 раз** — на 3-м цикле переходи к арбитражу.
