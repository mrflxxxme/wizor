# 01-CONTEXT-LOADING — JIT-протокол загрузки контекста

> **Цель:** каждый незагруженный файл — это свободный токен для реальной работы.

## Принцип: Just-In-Time

**Не грузи превентивно.** Грузи когда конкретно понадобилось.

```
ПЛОХО:
  Read charter (5 KB) + Read все ADR (~30 KB) + Read все контракты (~20 KB)
  → 55 KB загружено, 90% не использовано

ХОРОШО:
  Read charter (5 KB) → Read STATUS (2 KB) → Read phase-spec (4 KB)
  → при "см. ADR-0003": Read ADR-0003 (2 KB)
  → при "scorer.py": Grep "def score" scorer.py (0.5 KB)
  → 13.5 KB загружено, всё использовано
```

## Приоритеты (charter §9)

| Приоритет | Файлы | Когда |
|---|---|---|
| **P0** | `BUILD-CHARTER.md` + `STATUS.md` + `HANDOFF.md` | 1× за сессию, bootstrap |
| **P1** | Phase-spec активной фазы | Старт задачи |
| **P2** | ADR, на которые ссылается phase-spec | При встрече ссылки |
| **P3** | `MEMORY-INDEX.md` → grep конкретного файла | Поиск по теме |
| **skip** | `archive/`, закрытые риски, не-активные фазы | Никогда превентивно |

## Токен-бюджеты (ориентиры)

| Тип задачи | Загружать | Context для работы |
|---|---|---|
| Простой фикс (1 файл) | 5–10 KB | 90–95 KB |
| Один endpoint | 10–20 KB | 80–90 KB |
| Фаза целиком | 30–60 KB | 40–70 KB |
| Cross-phase интеграция | 60–120 KB | делегируй суб-агенту |

**При приближении к 120 KB** → делегируй суб-агенту или сохрани state в HANDOFF + открой новую сессию.

## Стратегии

### 1. Read с offset/limit
Для больших файлов (ADR, phase-spec > 200 строк) читай только нужную секцию:
```
Read(file, offset=80, limit=40)   # только нужный раздел
```

### 2. Grep до Read
Ищешь термин или символ — сначала Grep, потом Read нужного фрагмента:
```
Grep(pattern="def score_readiness", glob="**/*.py")
```

### 3. Head-summary первым
Любой файл > 2 KB имеет блок `<!-- HEAD-SUMMARY (≤500т): ... -->` в начале — читай его первым через Grep или Read(limit=5).

### 4. MEMORY-INDEX — входная точка для recall
Перед загрузкой нового файла смотри `.planning/MEMORY-INDEX.md`: там таблица `tag | topic | pointer | gist`. Если тема есть — грузи только тот файл по указателю.

### 5. Skip archive
Не читай `_session-context/archive/` и закрытые ADR (status: superseded) — только если явно нужна история ревизий.

## Расширение контекста при необходимости

1. Grep в уже загруженных файлах
2. Read с offset/limit нового файла
3. Full Read только если нет другого пути
4. Делегирование суб-агенту со свежим контекстом
5. Handoff + новая сессия

## Anti-patterns

- **Читать все ADR перед стартом** — ADR читается только при явной ссылке из phase-spec.
- **Повторный Read того же файла** — файл уже в контексте; перечитывай только если был Edit.
- **Grep с широким pattern** — `Grep("a")` бесполезен; используй конкретные символы/ключевые слова.
- **Читать полный `contracts/README.md`** — достаточно `contracts/<ctx>/README.md` нужного контекста.
