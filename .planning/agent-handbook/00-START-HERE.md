# 00-START-HERE — протокол работы ИИ-агента в WIZOR

Этот файл — единственный source-of-truth для **как работать**, а не **что строить** (продукт — в `.planning/PRD.md`).

## Что такое WIZOR

WIZOR — **AI Readiness Platform для РФ**: аудит → мониторинг → рекомендации → авто-имплементация → верификация видимости сайта в ИИ-поиске (ChatGPT / Perplexity / Алиса-Нейро / GigaChat).  
Главный moat — авто-применение правок на стороне сайта + покрытие RU-стека.  
Модель доступа: **read-only-first** — Tier 0 Instant Audit (no-auth) → Manual track → Auto track (премиум).  
Полное ТЗ: `.planning/PRD.md`. Харнесс: `.planning/_meta/BUILD-CHARTER.md`.

## Bootstrap (обязателен для любой нетривиальной задачи)

Читай ровно эти файлы **один раз за сессию** — в порядке:

| # | Файл | Зачем |
|---|---|---|
| 1 | `.planning/_meta/BUILD-CHARTER.md` | Все решения харнесса, ростер, 9-шаговый цикл, форматы |
| 2 | `.planning/STATUS.md` | Активная фаза, блокеры |
| 3 | `.planning/HANDOFF.md` | Что оставил предыдущий агент: state / in-progress / next |
| 4 | `.claude/agents/<role>/` | Своя папка роли (system-prompt + workflows) |
| 5 | `.planning/roadmap/PNN-<slug>.md` | Phase-spec активной фазы |
| 6 | Затронутые контракты | `.planning/contracts/<ctx>/README.md` — JIT |

**Итог bootstrap:** ~15–25 KB загружено, остальное — JIT. Подтверждение пользователю — **неявное**: действуй автономно.

## Золотые правила

1. **JIT loading.** Не грузи превентивно. Сначала `MEMORY-INDEX.md`, потом конкретный файл.
2. **Пост-аудит обязателен.** Каждая фаза проходит `auditor`/Opus на шаге 7 — без исключений.
3. **Memory-update обязателен.** После аудита — `memory-curator`/Haiku обновляет STATUS/HANDOFF/JOURNAL (шаг 8).
4. **Founder ревьюит только гейты фаз** (ADR-0017). Внутри фазы PR авто-мерджятся (CI + `reviewer` + `auditor` PASS); ИИ-агенты мерджат сами и могут спавнить доп-сессии (Agent tool / `claude -p`) для параллельных задач и управления контекстом.
5. **Перед PR — зелёный verify** (ADR-0018): self-run тесты + live-gold где возможно, evidence записан; live-gold невозможен → явный `deferred_live_gold` в гейт (не тихий пропуск). Автономность = подтверждённая результатами.
5. **Язык:** нарратив RU, идентификаторы / код / имена файлов — EN.
6. **Секреты — никогда в коде/коммитах.**
7. **≤2 ревью-цикла** у reviewer → потом эскалация к `architect` / founder.

## Anti-patterns

- **Загрузка всего `.planning/` сразу** — не делай. Bootstrap-5 + JIT.
- **Повторное чтение того же файла** — не нужно: файл уже в контексте.
- **Архитектурные решения без ADR** — нельзя. Нужен новый ADR + эскалация.
- **PR без exit-ritual** — блокирует мердж (см. `05-PR-WORKFLOW.md`).
- **Делегирование decision-making суб-агенту** — суб-агент предлагает, ты решаешь.
