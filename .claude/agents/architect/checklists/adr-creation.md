# adr-creation checklist (architect)

Запускать перед записью нового ADR.

- [ ] `id` уникален (следующий номер в `.planning/decisions/`)
- [ ] `status: proposed` → после консенсуса → `accepted`
- [ ] `date` = сегодня (ISO 8601)
- [ ] `supersedes: []` заполнен если заменяет старый ADR
- [ ] **Context** описывает проблему без решения
- [ ] **Decision** — одно конкретное решение (не список вариантов)
- [ ] **Consequences** включают как позитивные, так и негативные
- [ ] **Alternatives considered** — минимум 1 альтернатива с причиной отклонения
- [ ] ADR не противоречит существующим accepted ADR (проверить список)
- [ ] ADR не нарушает ни один из 10 инвариантов charter §6
- [ ] Эскалирующий агент уведомлён (хендофф с ссылкой на ADR)
